# 건강신호등 실행계획 — Action Plan / HOW

## 1. 운영 목표와 원칙

- 목적: 건강상태를 진단하는 것이 아니라 **상담이 먼저 필요한 학생을 빠르게 찾고 개입을 연결**한다.
- 대상: 건강정보 활용에 동의한 재학생. 1차 파일럿은 500명, 8주.
- 판단 단위: `unhealthy / at-risk / fit` 예측과 SHAP 상위 3개 요인. 원본 응답과 함께 상담사가 최종 확인한다.
- 금지: 모델 결과만으로 불이익·학사조치·보험 판단을 하지 않는다. 학생에게 확정적 질병명으로 표시하지 않는다.

## 3. 등급별 개입 프로토콜

### Red — unhealthy 우선

1. OOF에서 정한 `unhealthy` 임계값을 넘은 학생을 우선순위 큐에 넣는다.
2. 상담사 화면에는 라벨 대신 `우선 확인 필요`, 확률, SHAP 요인 3개, 최근 변화만 노출한다.
3. 10분 체크리스트: 수면시간·수면질·스트레스·활동 저하·학업부담·즉시위험 신호를 확인한다.
4. 즉시위험 신호가 있으면 대학 위기대응 매뉴얼로 전환한다. 없으면 72시간 내 일반상담을 예약한다.
5. 7일·14일에 재확인한다. 2회 연속 Red이면 담당 상담사가 개별 관리계획을 작성한다.

### Amber — at-risk 경계군

1. SHAP 요인에 따라 수면·스트레스·활동 중 **한 가지 핵심 미션**만 배정한다.
2. 수면: 취침·기상 고정, 7일 수면일지. 스트레스: 5분 호흡·상담 콘텐츠. 활동: 하루 20분 걷기.
3. 7일차 이행률 50% 미만이면 리마인드, 14일차에도 개선이 없거나 Red로 이동하면 상담 큐로 승격한다.

### Green — fit 유지군

1. 월 1회 셀프체크와 예방 콘텐츠를 제공한다.
2. 확률이 임계값 근처이거나 핵심 3개 컬럼 중 2개 이상 결측이면 Green 확정 대신 `재응답 요청`으로 보낸다.

## 4. 데이터·모델 구현 명세

- 입력 계약: 13개 원본 컬럼, 학생 식별자는 별도 암호화 키로 분리한다.
- 검증: 허용 범위, 범주 집합, 중복 ID, 결측률, 스키마 버전을 배치 전 검사한다.
- 추론: 학습 시점의 전처리기·클래스 순서·임계값을 하나의 버전으로 묶는다.
- 설명: 전체 SHAP 중요도는 정책 설계에, 개인 SHAP은 상담 질문 순서에만 사용한다.
- 로그: 모델 버전, 입력 스키마, 확률, 등급, SHAP 요인, 상담 결과, 동의 철회 여부를 남긴다.
- 접근권한: 상담사는 담당 학생, 데이터팀은 비식별 분석 데이터만 조회한다. 모든 조회는 감사로그를 남긴다.

## 5. 8주 파일럿과 KPI

| 구분 | 지표 | 목표/판정 | 측정법 |
|---|---|---|---|
| 모델 | unhealthy recall | 파일럿 기준선 이상 유지 | 상담 확인 라벨과 주간 비교 |
| 운영 | Red 24시간 접촉률 | 90% 이상 | 최초 접촉 타임스탬프 |
| 운영 | Red 72시간 상담연결률 | 70% 이상 | 예약·상담 완료 로그 |
| 효과 | 14일 등급 개선율 | 통상안내군 대비 +5%p 이상 | 즉시개입 250명 vs 통상안내 250명 |
| 부담 | 오탐 상담 비율 | 상담 수용량 안에서 관리 | 상담 후 `추가조치 불필요` 비율 |
| 형평 | 성별·학년별 recall 격차 | 10%p 이하 | 그룹별 주간 리포트 |
| 데이터 | 핵심 컬럼 결측률 | 20% 이하 | 배치 품질검사 |
| 안정성 | PSI | 0.2 이하 | 학습분포 vs 최근 4주 |

## 6. 중단·확대 기준

- 즉시 중단: 동의 없는 데이터 유입, 민감정보 노출, 클래스 매핑 오류, 잘못된 학생에게 알림 발송.
- 자동 확장 중단: 그룹 recall 격차 >10%p, 핵심 컬럼 결측률 >20%, PSI >0.2, 상담 수요가 주간 수용량의 110% 초과.
- 8주 후 확대: 심각한 안전사고 0건, 72시간 연결률 70% 이상, 등급 개선 +5%p 이상, 형평성 기준 충족 시 2,000명으로 확대한다.
- 재학습: 분기 1회가 기본. 성능·분포 경보가 발생하면 데이터 품질과 라벨 정의를 먼저 점검한 뒤 재학습한다.

## 7. 발표에서 보여줄 연결고리

`EDA의 핵심 3요인 → Balanced Accuracy 기반 모델 → SHAP 개인 설명 → 등급별 개입 → 14일 재측정 → KPI로 효과 검증`

이 흐름을 한 장의 서비스 블루프린트로 제시하고, 모델 점수보다 **누가·언제·무엇을·어떻게 실행하는지**를 발표의 결론으로 둔다.
